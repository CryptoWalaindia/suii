Nodes, Network & Protocol  —  10 TERMS

01
BITCOIN NODE
A Bitcoin node is a computer running Bitcoin software. It connects to the Bitcoin network,checks transactions and blocks, and helps make sure Bitcoin’s rules are followed.
→
02
CONSENSUS MECHANISM
A consensus mechanism is the method a network uses to agree on what is valid. In Bitcoin,consensus depends on rules, nodes, proof of work, and the longest valid chain.
→
03
DECENTRALIZATION
Decentralization means control is spread across many participants instead of being held by onecentral authority. In Bitcoin, decentralization means no single company, government, or servercontrols the
→
04
DOUBLE SPEND
Double spend means trying to spend the same digital money more than once. Bitcoin wasdesigned to solve this problem without needing a central authority.
→
05
FULL NODE
A full node is a Bitcoin node that independently verifies the entire set of Bitcoin rules. It checkstransactions, blocks, and the blockchain history without relying fully on someone else.
→
06
MEMPOOL
The mempool is a waiting area for unconfirmed Bitcoin transactions. When a transaction isbroadcast but not yet included in a block, it usually waits in the mempool.
→
07
PEER-TO-PEER NETWORK
A peer-to-peer network is a system where computers communicate directly with each otherinstead of depending on one central server. Bitcoin uses a peer-to-peer network to sharetransactions and blocks.
→
08
TIMECHAIN
Timechain is a Bitcoin-native term used to describe Bitcoin’s ordered chain of blocks. Ithighlights that Bitcoin records transactions in a time-based sequence.
→
09
TRANSACTION FEE
A Bitcoin transaction fee is a small amount paid by the sender to encourage miners to includethe transaction in a block. Fees can rise or fall depending on network activity and demand forblock space.
→
10
UTXO
UTXO stands for unspent transaction output. It is a piece of Bitcoin that has been received butnot yet spent. Bitcoin uses UTXOs to track ownership instead of normal account balances.
→
